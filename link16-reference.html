Gaps & Limitations

This reconstruction is fundamentally limited by what is publicly available.

What We Do NOT Have

  • DFI/DUI Definitions: The Data Field Identifier / Data Use Identifier system that defines exact bit-level field layouts within each J-series message word is contained in MIL-STD-6016 and STANAG 5516 — both CONTROLLED/RESTRICTED documents not available to the public.
  • Complete Field Layouts: For the vast majority of messages (61 of 69), we have only the message name and NPG association. Exact bit positions, field widths, encoding schemes, and value enumerations are unknown.
  • Continuation/Extension Words: While we know the word format structure (Initial/Continuation/Extension), the specific field layouts of C and E words for each message type are not publicly documented.
  • Network Entry Protocol Details: The precise handshake sequences, challenge/response formats, and timing for JTIDS network entry are controlled.
  • Crypto Variables: TSEC/MSEC key formats, CVLL assignment procedures, and rekeying protocols are classified.
  • JREAP Encapsulation Details: While JREAP-A/B/C existence is documented, detailed PDU formats beyond basic framing are not fully public.
  • Full NPG-to-Message Mapping: The complete assignment of messages to NPGs (especially for weapons coordination and national messages) is only partially available.
  • Messages J4.x, J18-27.x, J29-30.x: These label ranges appear unused or reserved, but we cannot confirm this without access to the standard.

What We Do Have (and Confidence Levels)

  • Physical Layer: HIGH confidence — frequency band, modulation, TDMA timing are well-documented in public literature.
  • Link Layer Framing: HIGH confidence — word structure, header format, packing modes from SISO-STD-002 and Wireshark dissector.
  • Message Names & NPGs: HIGH confidence — 69 message types enumerated in the Wireshark dissector (DSTO-TN-1257).
  • PPLI & Track Fields: LOW-MEDIUM confidence — partial field names inferred from simulation papers, flight sim docs, and the AFIT thesis. Bit offsets mostly unknown.
  • Simulation Header: HIGH confidence — fully specified in SISO-STD-002-2021.

Methodology

This document was built by:

  1. Extracting text from SISO-STD-002-2021, DSTO-TN-1257, and the AFIT IP-over-Link-16 thesis
  2. Parsing the Wireshark dissector source code for message type enumerations
  3. Cross-referencing web sources (Wikipedia, Signal ID Wiki, Grokipedia, flight sim forums)
  4. Assigning confidence levels based on source quality and cross-reference count
  5. Generating an intermediate JSON schema, then this HTML reference from it
